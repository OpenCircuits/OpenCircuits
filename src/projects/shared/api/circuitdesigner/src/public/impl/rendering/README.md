# Circuit Designer Rendering (`public/impl/rendering`)

This directory provides the lower-level abstraction layer that bridges the gap between the purely mathematical/logical circuit primitives and actual visual pixels on an HTML canvas.

## Files

- **`RenderHelper.ts`**: An abstraction wrapper around the native `CanvasRenderingContext2D`. It provides a streamlined, type-safe API for executing drawing commands (like `moveTo`, `lineTo`, and applying `Style` objects) while managing the canvas context state.
- **`RenderScheduler.ts`**: A performance-focused utility class that batches render requests. It ensures the canvas is only painted once per browser paint cycle (using `requestAnimationFrame`), preventing redundant and expensive draw calls during rapid state changes.
- **`CanvasTextMeasurer.ts`**: A utility class to accurately measure the pixel dimensions of text given a specific font style, allowing the designer to size boxes and labels dynamically.
- **`RenderState.ts`**: Maintains state specific to the current rendering frame.

## Sub-Directories

- **`renderers/`**: Contains the specific classes responsible for drawing discrete elements of the viewport, such as the background grid, debugging bounds, and the actual geometric primitives (`Prim`s) generated by the `assembly` layer.
